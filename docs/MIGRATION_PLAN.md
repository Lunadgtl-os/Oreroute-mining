# Ore Route migration plan

## Phase 1 — independent interface

- Establish the GitHub-owned React application
- Apply the approved mining visual identity
- Recreate the core navigation and operational views
- Add automated build validation and deployment

## Phase 2 — production data layer

- Select the independent backend and authentication provider
- Implement the 10 Base44 operational entities in a versioned database schema
- Add organisation-based permissions and audit logging
- Replace representative records with authenticated API queries

## Phase 3 — workflow parity

- Material passport creation and lifecycle controls
- Custody-event evidence capture
- Shipment dispatch, checkpoint and receiving workflows
- Transport tender bidding and award controls
- Wash-plant input/output lineage
- Trade dossier and payment-milestone controls
- Evidence-vault permissions and verification

## Phase 4 — production hardening

- Security review
- Data migration rehearsal
- Observability and exception monitoring
- Backups and recovery testing
- User-acceptance testing before cutover
